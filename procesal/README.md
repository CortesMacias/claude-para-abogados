# Procesal — Derecho procesal

Este módulo asiste en la preparación y gestión de procedimientos judiciales en todas las jurisdicciones: civil, penal, contencioso-administrativo y social. Cubre la elaboración de cronologías de hechos, borradores de demanda, análisis de requerimientos recibidos, control de plazos procesales, intake de nuevos asuntos, briefings ejecutivos y visión de portfolio. Pensado para litigantes, procuradores y departamentos de contencioso.

## Skills disponibles

| Skill | Comando | Descripción |
|-------|---------|-------------|
| Cronología | `/cronologia` | Genera una línea temporal de hechos relevantes a partir de documentos del caso |
| Demanda | `/demanda` | Prepara un borrador estructurado de demanda con hechos, fundamentos y petitum |
| Requerimiento recibido | `/requerimiento-recibido` | Analiza un requerimiento o demanda recibida e identifica plazos y opciones de respuesta |
| Plazos | `/plazos` | Calcula y controla plazos procesales según jurisdicción y tipo de procedimiento |
| Intake | `/intake` | Estructura la información inicial de un nuevo asunto litigioso |
| Briefing | `/briefing` | Genera un resumen ejecutivo del estado del procedimiento para cliente o socio |
| Portfolio | `/portfolio` | Vista general del estado de todos los asuntos litigiosos activos |

## Primeros pasos

1. Ejecuta la entrevista inicial del proyecto para configurar tu perfil y preferencias.
2. Prueba `/cronologia` con la documentación de un caso para generar la línea temporal de hechos.
3. Usa `/plazos` para verificar los plazos procesales de un procedimiento en curso.
4. Ejecuta `/intake` cuando recibas un nuevo asunto para estructurar toda la información desde el primer día.

## Casos de uso típicos

- Preparación de la cronología de hechos para un juicio ordinario civil.
- Análisis de una demanda recibida para identificar plazos de contestación y posibles excepciones.
- Control centralizado de plazos procesales de todos los asuntos del despacho.
- Generación de un briefing para el cliente sobre el estado de su procedimiento.

## Lo que este plugin no hace

- **No presenta escritos ante juzgados**: no firma ni envía documentos procesales a través de LexNET ni ningún otro sistema.
- **No sustituye al procurador**: no asume funciones de representación procesal.
- **No calcula costas procesales**: no emite tasaciones de costas ni minutas de honorarios.
- **No sustituye el criterio profesional** del abogado: los borradores deben ser revisados y firmados por un letrado colegiado.
