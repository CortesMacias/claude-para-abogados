# Regulatorio — Cumplimiento regulatorio

Este módulo asiste en el seguimiento y cumplimiento de normativa sectorial y administrativa. Permite comparar versiones de normas, identificar brechas de cumplimiento, redactar documentos de adaptación normativa y configurar alertas ante cambios regulatorios relevantes. Orientado a departamentos de compliance, asesorías jurídicas de empresas reguladas y despachos especializados en sectores como telecomunicaciones, energía, financiero o sanitario.

## Skills disponibles

| Skill | Comando | Descripción |
|-------|---------|-------------|
| Diff normativo | `/diff` | Compara dos versiones de una norma o reglamento y resalta los cambios relevantes |
| Gaps | `/gaps` | Identifica brechas entre la situación actual de la organización y los requisitos normativos |
| Redacción | `/redaccion` | Genera borradores de políticas, procedimientos o informes de adaptación normativa |
| Alertas | `/alertas` | Configura el seguimiento de cambios normativos en áreas de interés |

## Primeros pasos

1. Ejecuta la entrevista inicial del proyecto para configurar tu perfil y preferencias.
2. Prueba `/diff` con dos versiones de una norma sectorial para ver los cambios destacados.
3. Usa `/gaps` para auditar el nivel de cumplimiento de tu organización frente a una normativa concreta.
4. Configura `/alertas` para recibir notificaciones cuando se modifiquen las normas de tu sector.

## Casos de uso típicos

- Comparación de la versión vigente y la reforma de una ley sectorial para identificar nuevas obligaciones.
- Análisis de brechas de cumplimiento frente al nuevo marco regulatorio de servicios digitales.
- Redacción de un procedimiento interno de adaptación a una circular del supervisor sectorial.
- Seguimiento de cambios normativos en el BOE y el DOUE para áreas de práctica seleccionadas.

## Legislación de referencia

- **LPAC** — Ley 39/2015, de 1 de octubre, del Procedimiento Administrativo Común de las Administraciones Públicas
- **Normativa sectorial** aplicable según el ámbito de actividad (telecomunicaciones, energía, financiero, sanitario, farmacéutico, etc.)

## Lo que este plugin no hace

- **No realiza actividades de lobby**: no contacta con reguladores ni gestiona relaciones institucionales.
- **No envía comunicaciones formales a supervisores**: no presenta escritos ante la CNMC, CNMV, Banco de España ni otros organismos reguladores.
- **No interpreta normativa técnica**: no analiza normas UNE, ISO ni estándares técnicos no jurídicos.
- **No sustituye el criterio profesional** del abogado: las recomendaciones deben ser validadas por un profesional cualificado.
